# Price Battle: Симулятор конкурентного ценообразования

## 1. Описание

**Price Battle** — это программный комплекс, предназначенный для автоматизации и оптимизации стратегий ценообразования в условиях жесткой конкуренции.

**Предметная область:** B2B и B2C рынки, где поведение конкурентов напрямую влияет на итоговую прибыль компании (например, SaaS-решения, агрегаторы такси, авиаперевозки).
**Цель проекта:** Создание надежного математического инструмента, который предсказывает оптимальное поведение продавцов на основе Теории игр и гарантирует выполнение бизнес-ограничений (покрытие себестоимости, целевая маржа) с помощью Линейного программирования.

---

## 2. Математическая модель

Проект опирается на две ключевые математические концепции:

### Теория игр (Поиск равновесия Нэша)

Взаимодействие компаний моделируется как матричная игра. Алгоритм ищет равновесие Нэша в чистых или смешанных стратегиях с использованием метода перебора носителей (Support Enumeration).
Для платежной матрицы $A$ и стратегий игроков $x$ (вектор вероятностей первого игрока) и $y$ (вектор вероятностей второго игрока) ожидаемая прибыль вычисляется как:
$$E = x^T A y$$
При условии, что $\sum x_i = 1$ и $\sum y_j = 1$.

### Линейное программирование (Оптимизация цены)

После определения стратегии, система рассчитывает конкретную цену продукта, максимизируя прибыль с учетом заданных ограничений (себестоимость, требуемая маржа, психологический потолок рынка).
Целевая функция максимизации прибыли с единицы товара:
$$Z = price - cost \to \max$$
При системе ограничений:

1. Ограничение минимальной цены (защита маржи): $price \ge cost \cdot \left(1 + \frac{min\_margin}{100}\right)$
2. Ограничение максимальной цены (рыночный спрос): $price \le max\_price$

## 3. Технологический стек

* **Язык программирования:** Python 3.10+
* **Математическое ядро:** `numpy`, `nashpy` (Теория игр), `pulp` (Линейное программирование)
* **REST API и Валидация:** `FastAPI`, `uvicorn`, `Pydantic`
* **Тестирование:** `pytest`
* **Конфигурация:** `PyYAML`

---

## 4. Быстрый старт

**1. Клонирование репозитория и установка зависимостей:**

```bash
git clone <url_репозитория>
cd price_battle
python -m venv venv
source venv/bin/activate  # Для Windows: venv\Scripts\activate
pip install -r requirements.txt
```

**2. Запуск через консоль (CLI) с использованием сценариев:**

```bash
python main.py --config config/scenario.yaml typical
```

**3. Запуск REST API сервера:**

```bash
python main.py --api
# Документация Swagger будет доступна по адресу http://127.0.0.1:8000/docs
```

## 6. Тестирование

Проект покрыт unit-тестами с использованием фреймворка pytest. Проверяются как позитивные, так и негативные сценарии работы алгоритмов, Pydantic-валидация и перехват системных исключений с помощью Mock-объектов.

Для запуска всех тестов выполните команду:

```bash
pytest -v
```

## 7. Описание сценариев

Все бизнес-ситуации описаны в файле config/scenario.yaml. Доступные сценарии для тестирования:

* **`typical` (Умеренная конкуренция)**:
    Конкуренция провайдеров облачной инфраструктуры (SaaS B2B). Базовая матрица 2x2. Показывает штатную работу системы, находя оптимальную смешанную стратегию и стабильную маржу.

* **`aggressive` (Жесткая ценовая война)**:
    Война агрегаторов такси за водителей и пассажиров. Расширенная матрица 3x3 без чистого равновесия (нет седловой точки). Алгоритм подбирает вероятностное распределение между агрессивными скидками, стандартом и премиями водителям.

* **`infeasible` (Конфликт ограничений - Edge Case)**:
    Рынок спешелти-кофе в условиях инфляции. Сценарий, где желаемая минимальная маржа конфликтует с психологическим максимумом рынка. Система корректно перехватывает сбой и возвращает статус "Конфликт ограничений".

* **`stress` (Стресс-тест вычислительного ядра)**:
    Ценовые войны авиакомпаний. Сложная матрица 4x4 с множественными вариантами демпинга и промо-тарифов. Проверяет стабильность работы алгоритма перебора носителей (Nashpy) при значительном росте размерности данных.
